Motion Principles

Our motion design is guided by three core principles that create a cohesive, polished experience.

Purposeful

Every animation has a clear purposeâ€”guiding attention, providing feedback, or showing relationships between elements.

Refined

Subtle, sophisticated motion that enhances rather than distracts. Nothing excessive or playful.

Responsive

Fast enough to feel immediate, slow enough to be perceived. Animation respects the user's time.

Easing Curves

Easing curves define how animations accelerate and decelerate. Click "Play" to see each curve in action.

Ease Out

cubic-bezier(0, 0, 0.2, 1)

Starts fast, ends slow. Best for elements entering the screen or appearing.

Use for: Modals appearing, dropdowns opening, tooltips

Ease In

cubic-bezier(0.4, 0, 1, 1)

Starts slow, ends fast. Best for elements leaving the screen or disappearing.

Use for: Modals closing, elements being dismissed

Ease In-Out

cubic-bezier(0.4, 0, 0.2, 1)

Smooth acceleration and deceleration. Best for transitions and transformations.

Use for: Element movement, state changes, transforms

Linear

linear

Constant speed throughout. Use sparinglyâ€”feels mechanical.

Use for: Loading spinners, progress bars (continuous motion)

Duration

Animation duration should match the distance traveled and importance of the change.

Fast (150ms)

Small changes, hover states, micro-interactions

Normal (250ms)

Default for most transitions, state changes

Slow (350ms)

Best Practices

âœ“ Do

  • Keep animations under 400ms
  • Use ease-out for entering elements
  • Animate one property at a time when possible
  • Respect user's motion preferences
  • Make hover states immediate (150ms or less)

âœ— Don't

  • Don't animate multiple elements simultaneously
  • Don't use bounce or elastic easing
  • Don't animate on every interaction
  • Don't use long durations (>500ms)
  • Don't animate layout properties excessively
